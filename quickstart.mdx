---

This guide walks you through everything you need to get EZAnim running in your Roblox game â€” from grabbing the module to playing and stopping your first animation on a character. The whole process takes less than five minutes.

<Note>
  Every model you pass to EZAnim must contain a `Humanoid`. If the `Humanoid` does not already have an `Animator` child, EZAnim will create one for you automatically.
</Note>

<Steps>
  <Step title="Get the module">
    Open **Roblox Studio** and navigate to the **Creator Store** (previously known as the Toolbox). Search for **EZAnim** and click **Get** to add it to your inventory. Once it appears in your inventory, insert it into your game.

    Place the `EZAnim` `ModuleScript` in either `ServerStorage` (for server-side animation control) or `ReplicatedStorage` (if you need both the server and the client to access it). `ServerStorage` is the recommended default for most character animation workflows.
  </Step>

  <Step title="Require the module">
    In the `Script` or `LocalScript` where you want to control animations, require EZAnim to get the `AnimationService` table:

    ```lua
    local AnimationService = require(game.ServerStorage.EZAnim)
    ```

    If you placed EZAnim in `ReplicatedStorage`, swap the path accordingly:

    ```lua
    local AnimationService = require(game.ReplicatedStorage.EZAnim)
    ```
  </Step>

  <Step title="Initialize with your animation list">
    Call `AnimationService.init()` and pass a `List` table that maps friendly string keys to your Roblox animation asset IDs. The returned `Anim` object is your live service instance â€” keep a reference to it for all subsequent calls.

    ```lua
    local Anim = AnimationService.init({
        List = {
            Idle   = "rbxassetid://YOUR_IDLE_ID",
            Walk   = "rbxassetid://YOUR_WALK_ID",
            Jump   = "rbxassetid://YOUR_JUMP_ID",
            Attack = "rbxassetid://YOUR_ATTACK_ID",
        }
    })
    ```

    Replace each `YOUR_*_ID` placeholder with the numeric asset ID of the corresponding animation you uploaded to Roblox. The keys (`"Idle"`, `"Walk"`, etc.) are the names you will use in every other EZAnim method call.

    <Tip>
      Store the `Anim` instance inside its own `ModuleScript` and `require` that module from every script that needs animation control. This ensures all scripts share the same service instance and the same internal track cache, preventing duplicate `AnimationTrack` objects from being loaded for the same model.
    </Tip>
  </Step>

  <Step title="Play your first animation">
    Pass the character model, the animation key, the priority level, and whether the animation should loop to `Anim:play()`:

    ```lua
    local character = player.Character
    Anim:play(character, "Idle", "Idle", true)
    ```

    The third argument (`"Idle"`) is the priority tier â€” valid values are `"Idle"`, `"Movement"`, `"Action"`, `"Action2"`, `"Action3"`, and `"Action4"`. The fourth argument (`true`) tells EZAnim to loop the track. Setting it to `false` plays the animation once and stops automatically.
  </Step>

  <Step title="Stop the animation">
    When you no longer need the animation â€” for example when the player starts walking â€” call `Anim:stop()` with the character and the animation key:

    ```lua
    Anim:stop(character, "Idle")
    ```

    You can also pass an optional `fadeTime` (in seconds) as a third argument to control how quickly the track blends out:

    ```lua
    Anim:stop(character, "Idle", 0.3)
    ```

    To stop every animation playing on a model at once, use `stopAll`:

    ```lua
    Anim:stopAll(character)
    ```
  </Step>
</Steps>
