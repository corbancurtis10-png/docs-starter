`queue()` appends an animation ID to a model's FIFO queue. When the currently playing non-looping track fires its `Stopped` signal, the service automatically dequeues the next ID and plays it — letting you chain animations without writing manual `Stopped` listeners. Calling `queue()` alone does **not** start playback; you must call `play()` on the first animation yourself to get the chain running.

## Signature

```lua
AnimationService:queue(model: Model, id: string): nil
```

## Parameters

<ParamField body="model" type="Model" required>
  The character or object whose animation queue you want to append to. The model must have already been passed to `init()`.
</ParamField>

<ParamField body="id" type="string" required>
  The animation ID string to enqueue. Must be a key that exists in the `List` you passed to `init()` for this model.
</ParamField>

## Returns

`nil`

## Example

The following example plays a three-hit melee combo. `Attack1` starts immediately via `play()`; `Attack2` and `Attack3` are queued and play sequentially as each previous track finishes.

```lua
local AnimationService = require(game.ReplicatedStorage.EZAnim)

-- Initialise the model with all three attack animations
AnimationService:init(character, {
  Attack1 = "rbxassetid://111111111",
  Attack2 = "rbxassetid://222222222",
  Attack3 = "rbxassetid://333333333",
})

-- Start the first hit immediately, then enqueue the follow-ups
AnimationService:play(character, "Attack1")
AnimationService:queue(character, "Attack2")
AnimationService:queue(character, "Attack3")
```

<Note>
  `queue()` only appends to the queue — it does **not** start playback on its own. You must call `play()` on the first animation to kick off the chain. Once that non-looping track's `Stopped` signal fires, the service will automatically dequeue and play the next entry.
</Note>

<Note>
  `queue()` only works with **non-looping** animations. Looping tracks never fire `Stopped`, so the queue is never drained and subsequent entries will not play.
</Note>

<Tip>
  Call `reset()` to clear the queue mid-combo — for example, if the player gets interrupted or you need to cancel the remaining chain.
</Tip>
