`AnimationService:resume()` restores a previously paused animation to its original playback speed, causing the animation to continue seamlessly from the frame at which it was frozen. No blending or reloading occurs — the track simply starts moving again. If `pause()` was never called beforehand, `resume()` sets the speed to `1` rather than throwing an error, so calling it on an already-playing or unpaused track is safe.

## Signature

```lua
AnimationService:resume(
    model: Model,
    id:    string
): nil
```

## Parameters

<ParamField body="model" type="Model" required>
  The Roblox `Model` instance whose animation you want to resume.
</ParamField>

<ParamField body="id" type="string" required>
  The key of the animation to resume, as registered in your `init()` list.
  The track must be currently loaded (i.e. `play()` must have been called
  for this `id` on this model).
</ParamField>

## Returns

`nil`.

## Example

```lua
local AnimService = require(game.ServerScriptService.AnimationService)

-- Pause during a cutscene
AnimService:play(character, "Idle", "Idle", true)
AnimService:pause(character, "Idle")

task.wait(3) -- cutscene runs for 3 seconds

-- Resume exactly where the idle was frozen
AnimService:resume(character, "Idle")
```

<Note>
  If `pause()` was never called before `resume()`, EZAnim falls back to a
  speed of `1`, so the animation will play at normal speed rather than
  throwing an error. This means calling `resume()` on a track that was never
  paused is safe but will snap it to speed `1` if it was playing at a
  different speed.
</Note>
