`getState()` retrieves the state string most recently stored against a model by `setState()`. Use it anywhere in your scripts to read the model's current animation state and make decisions — such as whether to block a new animation, trigger a sound effect, or update a UI element. If no state has ever been set on this model, or if `reset()` has been called since the last `setState()`, the method returns `nil`.

## Signature

```lua
AnimationService:getState(model: Model): string | nil
```

## Parameters

<ParamField body="model" type="Model" required>
  The character or object whose state you want to read. The model must have been initialised with `init()`.
</ParamField>

## Returns

A `string` containing the state name set by `setState()`, or `nil` if no state has been assigned to this model yet or if `reset()` has cleared it.

## Example

Read the current state before deciding which animation to play on a humanoid, and display it in a debug label.

```lua
local AnimationService = require(game.ReplicatedStorage.EZAnim)

-- Read state before playing an emote
local function tryPlayEmote(character, emoteId)
  local state = AnimationService:getState(character)

  if state == "Dead" or state == "Stunned" then
    print("Cannot play emote — character is:", state)
    return
  end

  AnimationService:play(character, emoteId)
end

-- Display state in a ScreenGui label (useful during development)
runService.Heartbeat:Connect(function()
  local state = AnimationService:getState(character)
  debugLabel.Text = "Anim State: " .. tostring(state)
end)
```

<Note>
  `getState()` returns `nil` in two situations: the model has never had a state assigned, or `reset()` was called after the last `setState()`. Always use `tostring()` if you intend to concatenate the result with a string.
</Note>
